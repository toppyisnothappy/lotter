# Specification: High-Performance POS Terminal

**Track ID:** pos-terminal_20260308
**Type:** Feature
**Created:** 2026-03-08
**Status:** Draft

## Summary

Implementing a high-speed Point of Sale (POS) user interface with sub-second item search, barcode scanning support, and cart checkout flow.

## Context

Lotter requires a robust POS interface for clerks to process daily retail transactions with maximum speed and reliability. High performance search and intuitive cart logic are essential.

## User Story

As a Retail Clerk, I want to quickly scan or search for items, add them to a cart, and process a checkout, so that I can handle customers efficiently without frustrating delays.

## Acceptance Criteria

- [ ] Clerks can search for items by name or SKU instantly.
- [ ] Barcode scanning input automatically adds an item to the cart or increments its quantity.
- [ ] The cart accurately calculates line-item totals and overall totals.
- [ ] Checkout flow correctly creates a transaction in the database.
- [ ] Upon checkout completion, stock quantities are reduced atomically.

## Dependencies

- **backend-migration_20260305**: Requires the `transactions` schema and stock decrement logic.
- **inventory-mgmt_20260305**: Relies on existing products to be searchable.

## Out of Scope

- Applying POS transactions to outstanding customer debt (that is for the CRM track).
- Receipt printing or email integration.
- Returns or refunds handling.

## Technical Notes

- Keep all search operations debounced and indexed on the backend.
- Cart state should be maintained on the client (Zustand or React state) to ensure immediate responsiveness until checkout.
- Barcode scanner typically acts as a keyboard that fires a sequence of digits and an "Enter" key; listen for rapid sequential typing that ends in "Enter".
