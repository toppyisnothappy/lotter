# Specification: Inventory Engine: Core Management
 
**Track ID:** inventory-mgmt_20260305
**Type:** Feature
**Created:** 2026-03-05
**Status:** Draft
 
## Summary
 
Implementing the core inventory management system for store owners, supporting full CRUD operations for products with SKU tracking and stock management within the MySQL/Drizzle architecture.
 
## Context
 
As part of Phase 2, this track provides the base inventory infrastructure that store owners need before they can start making sales in the POS terminal.
 
## User Story
 
As a Store Owner, I want to manage my inventory and customer profiles so that I can prepare for sales and track outstanding credit.
 
## Acceptance Criteria
 
- [ ] Store owners can create new inventory items with name, SKU, price, and initial stock.
- [ ] Store owners can edit existing product details and adjust stock levels.
- [ ] Store owners can delete products from their inventory.
- [ ] All inventory operations are isolated by `organization_id`.
- [ ] Stock levels are correctly reflected in the database after manual adjustments.
 
## Dependencies
 
- **backend-migration_20260305**: Depends on the MySQL schema for `products` and the Drizzle connection established in the migration track.
 
## Out of Scope
 
- Barcode scanning integration.
- Bulk CSV import/export.
- Automated low-stock notifications (to be handled in a later polish track).
 
## Technical Notes
 
- Must strictly follow **Feature-Sliced Design (FSD)** layering (Entities for data, Features for CRUD actions).
- Use **Next.js Server Actions** for all data mutations.
- Ensure the `organization_id` is automatically applied to all queries via a shared middleware or server-side utility.
