# Specification: Multi-Tenant Staff Management & Auth

**Track ID:** multi-tenant-auth_20260308
**Type:** Feature
**Created:** 2026-03-08
**Status:** Draft

## Summary

Implementing Auth.js (NextAuth) with a MySQL adapter, Role-Based Access Control (RBAC), and multi-tenant organization switching to fully replace previous Supabase implementations.

## Context

Lotter has migrated its data layer to MySQL via Drizzle ORM. We must replace the legacy Supabase authentication flows with Auth.js, ensuring secure login, session management, and role-based permissions (Store Owner vs. Clerk) across different organization tenants.

## User Story

As a Store Owner, I want to authenticate securely, create and manage my store organization, and invite clerks with restricted permissions so that my data remains secure and employees can access only what they need.

## Acceptance Criteria

- [ ] Users can securely log in and generate an Auth.js session stored in MySQL.
- [ ] Users can create an Organization and be assigned the "Owner" role.
- [ ] Owners can invite/create "Clerk" accounts for their Organization.
- [ ] Clerks can only access the POS interface, while Owners can access Inventory, CRM, and Dashboard.
- [ ] All API routes and Server Actions enforce strict auth and organization-level isolation.
- [ ] Supabase references are completely removed from the auth lifecycle.

## Dependencies

- **backend-migration_20260305**: Requires the NextAuth MySQL tables (users, sessions, accounts) defined via Drizzle.

## Out of Scope

- User profile image uploads.
- Social (OAuth) logins—we will focus on standard credentials/magic links for now.
- Automated billing or subscription tiers based on organization size.

## Technical Notes

- Use the `@auth/drizzle-adapter`.
- Middleware should be updated to protect routes (`/admin`, `/[slug]/pos`, etc.) using Auth.js.
- Ensure the `organization_id` context is smoothly resolved for all authenticated requests.
