# Specification: Installment Payments & LINE Reminders

**Track ID:** installment-reminders_20260315
**Type:** Feature
**Created:** 2026-03-15
**Status:** Draft

## Summary

Implement an "Installment" payment method in the POS system that allows customers to pay in parts or defer payment. This includes tracking outstanding balances in the CRM and providing functionality for store owners to send payment reminders via LINE Messaging API.

## Context

Small retail businesses often provide credit or installment plans to trusted customers. Currently, our POS only handles full payments (Cash/Card). To support this, we need to link POS transactions to CRM identities and automate the follow-up process via social messaging.

## User Story

- **As a customer**, I want to pay for my items via installments so that I can manage my cash flow, and I want to receive clear reminders on LINE so I don't forget my due dates.
- **As a store owner**, I want to see a clear list of all outstanding installments and push official reminders to customers' LINE accounts with one click.

## Acceptance Criteria

- [ ] **POS Customer Picker**: Ability to search and select a registered customer during the POS checkout flow.
- [ ] **Installment Method**: A new "Installment" payment option in the `CheckoutModal`.
- [ ] **Debt Persistence**: Finalizing an installment sale must automatically increase the customer's `total_balance` in the database.
- [ ] **LINE Push Service**: Integration with LINE Messaging API (or a simulated robust utility) to send templated push messages.
- [ ] **Billing View**: A dashboard in the CRM showing:
    - Total amount of all installments.
    - Number of buyers with debt.
    - Buttons to "Send LINE Reminder" for individual customers.
- [ ] **Customer Insight**: A "Payment Trace" for customers showing their purchase history vs payment history.

## Out of Scope

- Recurring credit card billing (stripe subscriptions).
- Complex interest rate calculation (fixed debt only).
- Legal debt collection automation.

## Technical Notes

- The `processTransaction` server action needs to be updated to accept `customer_id` and handle the `payment_type: 'installment'`.
- Environment variables `LINE_CHANNEL_ACCESS_TOKEN` and `LINE_CHANNEL_SECRET` will be used for the messaging logic.
