# Specification: Customer LINE ID Integration

**Track ID:** customer-line-id_20260315
**Type:** Feature
**Created:** 2026-03-15
**Status:** Draft

## Summary

Expand the Customer CRM to support storing and managing LINE Account IDs for customers. This allows shop owners to keep track of social communication handles alongside traditional phone and email contact info.

## Context

In many regions, LINE is the primary communication channel between retailers and customers. Currently, the Lotter CRM only supports Phone and Email. Adding a LINE ID field is essential for local business operations.

## User Story

As a store owner, I want to keep the LINE account of a customer when adding a new customer or updating an existing one, so that I can easily reach out to them through their preferred social channel.

## Acceptance Criteria

- [ ] New `line_id` column in the `customers` database table.
- [ ] Updated `Customer` model and TypeScript interfaces to include `line_id`.
- [ ] "LINE Account ID" input field added to the `CustomerForm` (Registration and Edit).
- [ ] LINE ID displayed in the `CustomerTable` and `CustomerProfile` views.
- [ ] Validated that LINE ID is persisted correctly during create and update operations.

## Dependencies

- Existing `customers` table and CRM features.

## Out of Scope

- Automated LINE bot integration.
- Verification of LINE IDs via the LINE API.

## Technical Notes

- The `line_id` should be an optional `varchar(255)` field.
- Migration will be handled via Drizzle.
