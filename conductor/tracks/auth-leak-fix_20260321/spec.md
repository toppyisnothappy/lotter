# Specification: Auth & Route Protection Remediation (Bug Fix)

**Type:** Bug
**Created:** 2026-03-21

## Summary

Fix a security leak where protected store management pages (/pos, /inventory, /customers, /admin) are accessible without authentication.

## Context

The Lotter platform requires store owners and clerks to be authenticated before accessing tenant-specific data. Currently, the entire system is reported as accessible without a valid session.

## Acceptance Criteria

- [ ] Users attempting to access any page under /pos, /inventory, /customers, or /admin without being logged in are immediately redirected to /login.
- [ ] Authenticated users can access these pages according to their role (Owner vs Clerk).
- [ ] Middleware correctly handles edge cases like API routes and public assets.

## Steps to Reproduce

1. Close all sessions.
2. Navigate to http://localhost:3000/inventory.
3. System loads and displays inventory data instead of redirecting to login.

## Technical Considerations

- Inspect src/auth.config.ts and src/middleware.ts for logic errors.
- Verify that auth.user is being correctly populated from the database session.
- Ensure NEXTAUTH_URL and AUTH_SECRET are correctly configured.
